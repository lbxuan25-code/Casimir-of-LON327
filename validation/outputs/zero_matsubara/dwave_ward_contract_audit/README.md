# d-wave exact-static Ward contract audit

This directory is reserved for diagnostic outputs generated by

```bash
python -m validation.run_dwave_static_ward_contract_audit
```

The audit is fail-closed and never applies the static longitudinal projection. It
starts from the phase-1 analytic result that the exact periodic Brillouin-zone
target is

\[
u_L K_{\mathrm{eff}}=0,
\qquad
K_{\mathrm{eff}}u_R=0.
\]

For one finite quadrature it reports the exact algebraic decomposition

\[
u_L K_{\mathrm{eff}}
=R_S-C_\eta K_{\eta\eta}^{-1}K_{\eta S}+r_{\mathrm{primitive}},
\]

and its right-side counterpart. The terms are further split into:

- `R_S`: `equal_forward`, `-delta_v_mid`, and `qM_mid`;
- primitive closure: bubble/translation and direct/contact residuals;
- `C_eta`: EM-collective contraction, collective bubble rotation, and Goldstone
  counterterm rotation;
- Schur projection: the contribution induced by each collective source;
- LT diagnostics: the complex `K_0L`, `K_L0`, `K_LL`, `K_LT`, and `K_TL`
  entries and all source vectors divided by `|q|`.

A recommended first comparison is

```bash
OUT="validation/outputs/zero_matsubara/dwave_ward_contract_audit/raw"
mkdir -p "$OUT"

env \
  PYTHONUNBUFFERED=1 \
  OMP_NUM_THREADS=1 \
  OPENBLAS_NUM_THREADS=1 \
  MKL_NUM_THREADS=1 \
  NUMEXPR_NUM_THREADS=1 \
  python -m validation.run_dwave_static_ward_contract_audit \
    --orders xy yx \
    --outer-orders 80 96 128 \
    --epsabs 2e-4 \
    --epsrel 2e-2 \
    --inner-limit 60 \
    --max-point-evaluations 100000 \
    --quadrature gk15 \
    --norm max \
    --split-points 0 \
    --qx 0.03 \
    --qy 0.02 \
    --temperature-K 10 \
    --delta0-eV 0.1 \
    --eta-eV 1e-8 \
    --output "$OUT/dwave_q003_002_T10_ward_contract_o80_96_128.csv"
```

The CSV contains scalar convergence diagnostics. The JSON contains the full
complex vectors and component-level decompositions. Generated raw outputs remain
diagnostic artifacts and are not production references or valid Casimir inputs.
