# Resolved finite-q Matsubara inverse-Green Ward block

Status: resolved at the single-particle inverse-Green matrix level only.

This note freezes the analytic block identified by the diagnostic chain ending with
`debug_inverse_green_ward_audit.py`. It does not claim full response-level Ward closure and must not be used to mark the finite-q response as Casimir-ready.

## Scope

Resolved here:

```text
single-particle BdG inverse Green function
primitive A0/L/T electromagnetic vertices
phase_eta2 collective vertex
matrix-level finite-q Matsubara Ward generator
source/observable asymmetric primitive Ward vectors
```

Not resolved here:

```text
Kubo bubble assembly
contact/direct term
K_etaeta bubble plus counterterm
collective Schur completion
TM/TE Casimir input
```

The response-level primitive Ward audit shows that this block improves the response residual relative to the old real target-like vector, but does not close it. Therefore the remaining response-level mismatch must be decomposed separately.

## Definitions

Use the finite-q routing

```text
k_- = k - q/2
k_+ = k + q/2
```

and define

```text
G_-^{-1}(z_-,k_-) = z_- I - H_-(k_-)
G_+^{-1}(z_+,k_+) = z_+ I - H_+(k_+)
```

with the Matsubara transfer

```text
z_+ - z_- = i xi
xi = 2 pi n k_B T
```

The Nambu gauge generator is the density vertex

```text
Gamma_A0 = tau_3 = diag(+I, -I)
```

In the repository this is the `density_vertex` primitive.

The phase vertex used by the BdG finite-q code is Hermitian:

```text
Gamma_theta = [[0, i Delta], [-i Delta^dagger, 0]]
```

For the amplitude/phase collective coordinate used by the current TM/TE sandbox,

```text
eta2 = Delta0 * theta
Gamma_theta = Delta0 * Gamma_eta2
```

so a gauge phase variation expressed in the `eta2` coordinate carries a factor of `Delta0`.

## Observable-side identity

The observable-side, or row-side, Nambu Ward identity is

```text
G_+^{-1} tau_3 - tau_3 G_-^{-1}
  = + i xi Gamma_A0^obs
    + q Gamma_L^obs
    - 2 i Delta0 Gamma_eta2
```

For q along x, the primitive observable Ward vector is therefore

```text
u_L = (+i xi, +q, 0)
W_L  = (0, -2i Delta0)
```

where the primitive electromagnetic order is `(A0, L, T)` and the collective order is `(amplitude_eta1, phase_eta2)`.

This identity is the matrix-level form corresponding to the diagnostic candidate

```text
temporal_i_A0_real_L_imag_phase
```

against the diagnostic reference

```text
nambu_Gplus_tau_right_minus_tau_left_Gminus
```

in the inverse-Green Ward audit.

## Source-side identity

The source-side, or column-side, identity is the asymmetric counterpart

```text
tau_3 G_+^{-1} - G_-^{-1} tau_3
  = - ( - i xi Gamma_A0^src
        + q Gamma_L^src
        - 2 i Delta0 Gamma_eta2 )
```

Equivalently, as a response-level null direction one may use either

```text
nu_R = (-i xi, +q, 0)
W_R  = (0, -2i Delta0)
```

or the overall sign-flipped form

```text
nu_R = (+i xi, -q, 0)
W_R  = (0, +2i Delta0)
```

These two choices are the same null direction. They differ by an overall sign, so residual norms cannot distinguish them.

This identity is the matrix-level form corresponding to the diagnostic candidate

```text
temporal_minus_i_A0_real_L_imag_phase
```

against the diagnostic reference

```text
nambu_tau_left_Gplus_minus_Gminus_tau_right
```

in the inverse-Green Ward audit.

## Why the time coefficient is i xi, not xi

The inverse Green function is

```text
G^{-1}(z,k) = z I - H(k)
```

For Matsubara transfer,

```text
z_+ - z_- = i xi
```

Therefore the frequency part of

```text
G_+^{-1} tau_3 - tau_3 G_-^{-1}
```

is

```text
(z_+ - z_-) tau_3 = i xi tau_3 = i xi Gamma_A0
```

This fixes the time component of the primitive Ward vector on the imaginary axis. A real coefficient `xi Gamma_A0` corresponds to a real-transfer debug convention, not to the Matsubara response convention.

## Why tau3 sandwich is required

The BdG/Nambu gauge transformation is generated by `tau_3`:

```text
Psi -> exp(i tau_3 chi) Psi
Delta -> exp(2 i chi) Delta
```

Thus the matrix identity is not the plain difference

```text
G_+^{-1} - G_-^{-1}
```

and not the plain Hamiltonian difference

```text
H_+ - H_-
```

Instead, the Ward generator is inserted on the appropriate side:

```text
G_+^{-1} tau_3 - tau_3 G_-^{-1}
```

or

```text
tau_3 G_+^{-1} - G_-^{-1} tau_3
```

The earlier direct `Delta H` check is therefore expected to fail: it omits the Nambu gauge generator.

## Why the phase coefficient is -2i Delta0

The order parameter transforms as

```text
Delta -> exp(2 i chi) Delta
```

so at linear order

```text
delta Delta = 2 i chi Delta
```

The code's phase vertex is already Hermitian:

```text
Gamma_theta = [[0, i Delta], [-i Delta^dagger, 0]]
```

and the sandbox collective coordinate satisfies

```text
eta2 = Delta0 * theta
Gamma_theta = Delta0 * Gamma_eta2
```

Consequently the Ward generator in the `eta2` coordinate carries

```text
-2 i Delta0 Gamma_eta2
```

with the sign fixed by the inverse-Green matrix identity under the repository's current BdG and source/observable conventions.

The important point is that the factor of `i` is in the Ward coefficient. It is not hidden by treating `phase_eta2` as an anti-Hermitian vertex; in the current code convention `phase_eta2` is Hermitian.

## Why source and observable vectors are asymmetric

The finite-q BdG response metadata records the source/observable convention as

```text
J = (rho, -Vx, -Vy)
P = (rho, +Vx, +Vy)
```

Equivalently, density has the same left/right dagger sign, while spatial current has the opposite left/right dagger sign. Therefore a single real, symmetric target basis such as

```text
G  = xi A0 + q L
TM = -q A0 + xi L
```

cannot represent both the source-side and observable-side Matsubara Ward contractions.

The resolved matrix-level primitive vectors are instead asymmetric:

```text
observable side:
  nu_L = (+i xi, +q, 0)
  W_L  = (0, -2i Delta0)

source side:
  nu_R = (-i xi, +q, 0)
  W_R  = (0, -2i Delta0)
```

up to an overall sign on the right null vector.

## Diagnostic evidence that fixes this block

The inverse-Green Ward audit tested three representative k points. For the physical Matsubara transfer convention

```text
z_+ - z_- = i xi
```

it found machine-precision agreement:

```text
observable side:
  temporal_i_A0_real_L_imag_phase
  vs nambu_Gplus_tau_right_minus_tau_left_Gminus
  relative residual ~ 1e-14 to 1e-15

source side:
  temporal_minus_i_A0_real_L_imag_phase
  vs nambu_tau_left_Gplus_minus_Gminus_tau_right
  relative residual ~ 1e-14 to 1e-15
```

The real-transfer debug convention can also produce machine-precision matches for some real-coefficient candidates, but that is a control case with

```text
z_+ - z_- = xi
```

and is not the imaginary-axis Matsubara convention.

## Current response-level status

The primitive response-level Ward audit then tested whether this matrix-level block propagates to the response kernel. It found a clear improvement over the old real target-like vector, but not closure:

```text
matrix-inferred asymmetric primitive vector:
  K_eff residual ~ 2.37e-03
  K_eff residual / ||K_eff|| ~ 3.02e-03

real target-like vector:
  K_eff residual ~ 4.27e-03
  K_eff residual / ||K_eff|| ~ 5.44e-03
```

Therefore the real symmetric `G/TM` target basis is a genuine blocker, but not the only blocker.

The remaining mismatch must be decomposed across

```text
K_SS bubble/contact
K_Seta and K_etaS mixed blocks
K_etaeta bubble/counterterm
finite-q Kubo endpoint routing
Schur completion
```

before any production convention can be accepted.
